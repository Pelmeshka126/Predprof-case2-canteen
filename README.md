# Predprof 2025/2026 - Кейс №2 "Управление столовой"

MVP-веб-приложение для практического тура отборочного этапа (профиль "Информационные технологии").

## Что реализовано (обязательный минимум)

- Регистрация/авторизация пользователей.
- Роли: `student`, `cook`, `admin`.
- Функции ученика:
  - просмотр меню,
  - оплата (разовый платеж и абонемент, демо-режим),
  - отметка о получении питания,
  - отзывы о блюдах.
- Функции повара:
  - учет выданных блюд,
  - контроль остатков,
  - создание заявки на закупку.
- Функции администратора:
  - согласование/отклонение заявок,
  - сводная статистика,
  - отчет по питанию и затратам.
- Обработка исключений:
  - запрет повторной отметки питания,
  - ошибка при нехватке остатков.

## Стек

- Python 3.12
- Flask 3
- SQLite
- Jinja2 templates

## Быстрый запуск

```bash
python3 -m pip install --user --break-system-packages -r requirements.txt
python3 app.py
```

После старта:

- `http://127.0.0.1:5000/login`

Демо-аккаунты:

- `admin@predprof.local / admin123`
- `cook@predprof.local / cook123`
- `student@predprof.local / student123`

## Структура

- `app.py` - точка входа
- `app/__init__.py` - factory + инициализация
- `app/db.py` - схема БД + seed данных
- `app/auth.py` - логин/регистрация/доступ
- `app/routes.py` - бизнес-логика по ролям
- `app/templates/` - веб-интерфейс
- `docs/official_requirements.md` - требования из официальных источников

## Что показать на демо по регламенту кейса №2

1. Регистрация и вход.
2. Оплата (разово и абонемент).
3. Отметка получения питания.
4. Учет выдачи блюд поваром.
5. Создание заявки на закупку.
